# CLAUDE.md

Guidance for Claude Code instances working in this repo.

## What this project is

A local kanban board for browsing, organizing, and resuming Claude Code sessions across every repo the user has used `claude` in. Scans `~/.claude/projects/`, renders a sidebar of repos and a 6-column board per repo.

Stack: Node/Express backend (`server/`, port 3001) + React/Vite/Tailwind frontend (`frontend/`, port 5173). No database — all state lives in the filesystem or localStorage.

## The non-obvious architectural decision

**The app never writes to Claude Code's own files.** Column placement and renames are stored in a per-session sidecar `<sessionId>.meta.json` that this app created. Claude Code neither reads nor writes that file.

| File | Owner | This app |
| --- | --- | --- |
| `<sessionId>.jsonl` | Claude Code | **read only** |
| `<sessionId>.meta.json` | This app | read + write |
| `sessions-index.json` | Claude Code | **read only** |
| `~/.claude/settings.json` | Claude Code | read + merge (only `cleanupPeriodDays` is surfaced) |

Consequence: renames and kanban statuses are visible only in this app. `claude --resume` and the native Claude Code session picker don't see them. If a future task asks to "make rename visible in Claude Code," the user should be warned that Claude Code may overwrite edits to `sessions-index.json` or corrupt the `.jsonl`.

## Features and where they live

- **Repo list** — `GET /api/repos` enumerates `~/.claude/projects/` dirs containing ≥1 `.jsonl`. Each repo's real filesystem path is recovered via `getRepoPath()` in `server/index.js` (reads `originalPath` from `sessions-index.json`, falls back to the first user message's `cwd`, last resort: the dir name itself — which is lossy because Claude encodes slashes/dots/underscores all as `-`).
- **Board** — 6 columns defined in `frontend/src/types.ts` `COLUMNS`. Drag-drop uses `@hello-pangea/dnd`. On drop, `updateSessionMeta` writes the new status to the sidecar.
- **Resume** — `POST /api/sessions/:sessionId/resume` does **not** spawn a terminal and does **not** write any task files. It:
  1. Builds `claude --dangerously-skip-permissions --resume <id>` (flag gated by `skipPermissions`).
  2. Copies that string to the OS clipboard (`clip` on Windows, `pbcopy` on macOS, `xclip` on Linux).
  3. Runs `code "<repoPath>"` to open VS Code at the session's repo.
  4. Returns `{ command, repoPath, repoPathExists }`.
  The frontend shows the copied command in a bottom-right toast. User presses `` Ctrl+` `` and pastes. **Do not resurrect** the earlier `tasks.json` approach — the user explicitly rejected it.
- **AI rename** — `POST /api/sessions/:sessionId/summarize` reads every user + assistant text message from the session's `.jsonl`, concatenates them, sends to `claude-haiku-4-5-20251001` with a prompt asking for ≤5 words (max_tokens 30), and writes the result to `custom_name` in the sidecar. Anthropic API errors propagate with the real message (frontend toasts it).
- **Manual rename** — clicking the title on `SessionCard` enters inline edit. Enter/blur saves via `updateSessionMeta` with `custom_name`. Same sidecar as AI rename.
- **Settings** — `SettingsModal` handles: retention (`cleanupPeriodDays` in `~/.claude/settings.json`), resume command flag (localStorage), AI-rename word count (localStorage), semantic-index reconcile button + status tiles, hidden-repo substring patterns (localStorage).
- **Session details modal** — `SessionDetailsModal` fetches `GET /api/sessions/:sessionId/details` and renders tool counts, file groups (grouped by directory with read/edit/write tallies), the resumer brief section (button calls `POST /api/sessions/:sessionId/brief`), the key-decisions section (button calls `POST /api/sessions/:sessionId/decisions` with Sonnet, copy-markdown button for pasting into CLAUDE.md), and the lineage chain (`forkedFrom` parent + children found by scanning every `.jsonl` in the repo). Rendered via `createPortal(..., document.body)` to escape `@hello-pangea/dnd`'s transform wrapper which breaks `position: fixed`.
- **Resumer brief** — 2-sentence "where you left off" note generated by Haiku from the last ~12 turns. Saved to `.meta.json` under `brief`. Displayed on the card as an italic blockquote under the title whenever present.
- **Key decisions** — long-form markdown (Objective / Work done / Decisions / Learnings / Open questions) generated by Sonnet over the full transcript. Saved to `.meta.json` under `decisions`.
- **Weekly digest** — `POST /api/digest/weekly` takes `{ days, repoIds? }`. Walks every repo (or just the filtered subset), builds a compact per-session block from `.meta.json` + first/last message, feeds the pile to Sonnet with a first-person journal prompt. Button lives in the board header beside the stat tiles.
- **Semantic index** — hybrid BM25 + vector retrieval over every session. Built from `server/indexer/` (see below). Exposed via `POST /api/index/reconcile`, `POST /api/index/search`, `POST /api/index/ask`. UI entry points: Settings → Semantic index section (reconcile), magnifying-glass icon in the sidebar (Search + Ask modal).

## Running

```bash
npm run dev           # from root — runs both backend and frontend via concurrently
cd server && node index.js    # backend only
cd frontend && npm run dev    # frontend only (port 5173, proxies /api → 3001)
```

Backend loads `server/.env` via `dotenv`. `ANTHROPIC_API_KEY` is required for AI rename / brief / decisions / digest / ask. `AUTO_RECONCILE=true` runs the semantic-index reconcile on server startup; defaults to `false` so the user triggers it manually until trusted. Restart the backend after editing `.env`.

## Data shapes (don't break these)

`Session` (`frontend/src/types.ts`) — `{ sessionId, firstPrompt, summary, customName, status, messageCount, created, modified, gitBranch, brief, decisions }`. The backend returns this exact shape from `/api/repos/:repoId/sessions`. `brief` and `decisions` come from `.meta.json` and may be `null`.

`Repo` — `{ id, path, name, sessionCount }`. `id` is the raw directory name under `~/.claude/projects/` (slashes/dots/underscores encoded as `-`); `path` is the recovered real filesystem path; `name` is the last segment of `path`.

`COLUMN_CONFIG` — extended by the current "soft" design with `dot` (hex color) and `accent` (subtitle) fields. If you refactor components, preserve those additive fields or update every consumer.

## The semantic index (`server/indexer/`)

Fully offline hybrid retrieval layer. All components are in `server/indexer/` and the index persists to `~/.claude/vibekanban-index.json`.

- `embedder.js` — wraps `@huggingface/transformers`. Default model `Xenova/bge-small-en-v1.5` (384d, ~35 MB, quantized int8). Cached under `~/.claude/vibekanban-models/` via the `env.cacheDir` export. `embed(text)` returns a plain JS number array.
- `chunker.js` — `chunkSession(filePath, metaInfo)` returns `{ lineCount, sessionText, turns }`. Session-level text combines `custom_name`, `summary`, `brief`, and `firstPrompt`. Each turn is `User: ...\n\nAssistant: ...` capped at 8000 chars. `countLines` and `headHash` are used by the reconciler.
- `store.js` — flat JSON. Schema is `{ version, embedder, dim, sessions: { [sessionId]: { repoId, lastIndexedLineCount, lastIndexedTurnCount, headHash, indexedAt } }, chunks: [{ id, sessionId, repoId, kind, turnIndex?, text, vector, createdAt }] }`. Cached in-memory per-process.
- `bm25.js` — in-memory BM25 (k1=1.5, b=0.75). Built from the loaded index on first query via `buildBM25(index)`, invalidated by `invalidateBM25()` after reconcile. Tokenizer preserves `file/path.ts` and `ECONNRESET`-style tokens.
- `search.js` — `search(query, { k, mode, kind })` runs BM25 and/or cosine over the in-memory chunks, fuses with Reciprocal Rank Fusion (`c=60`). `retrieve(query, k)` builds the context block for Ask with numbered `[N]` headers.
- `reconcile.js` — diffs on-disk `.jsonl` files against the index. For each file: compare `lineCount` + `headHash` to stored state, produce action (`new` / `appended` / `rewritten` / `noop`), embed only the delta. Walks index entries with no matching file to tombstone them. Dry-run mode returns the plan without embedding.

**Gotchas here specifically:**
- **Line counts drive deltas, not byte offsets.** Robust against line-length variation.
- **`headHash` guards against rewrites.** If first 8 KB changed, full re-embed (rare; Claude Code is append-only so this is a safety net).
- **Forks create new files.** `claude --resume` writes a new `.jsonl` with `forkedFrom` metadata; the reconciler sees it as `new` — correct behavior.
- **Model download network-bound.** First reconcile on a cold machine waits on HuggingFace. If the user is offline and the cache is empty, the reconcile errors out. Frontend displays the error in the settings panel.
- **Don't rebuild BM25 inside every request.** The `invalidateBM25()` hook in `/api/index/reconcile` is the only place that should bust the cache.

## Design experiments (`designs/`)

`designs/` is gitignored (`.gitignore` has `designs/*`). It contains seven skill-based redesign variants produced in parallel by sub-agents. ## Gotchas

- **Directory-name encoding is lossy.** Always resolve real paths via `getRepoPath()`, never trust the raw dir name.
- **Dev script.** The root `npm run dev` uses `concurrently` and does not auto-reload the backend. After editing `server/index.js`, kill and restart (or switch to `node --watch index.js` if needed).
- **`.vscode/tasks.json` residue.** Earlier iterations wrote task files to user repos. If a user reports VS Code auto-running `claude --resume` on folder open, it's a stale file from the old behavior — search for `"Vibekanban: Resume Claude Session"` and delete the task or the whole file.
- **Windows clipboard.** `clip` is invoked via `spawn`; don't switch to `exec` with a piped echo — it trips quoting edge cases.
- **Modal portals.** Any modal that needs to fill the viewport (`SessionDetailsModal`, `WeeklyDigestModal`, `SearchModal`) must render via `createPortal(..., document.body)`. `@hello-pangea/dnd` applies `transform` to draggable wrappers which silently breaks `position: fixed` inside them.
- **Search vs Ask behavior.** Search does zero LLM calls; Ask hits Sonnet with the top-K chunks. If Ask shows only citations with no prose answer, the Sonnet call failed — check `/api/index/ask` response for the real error. Likely suspects: missing/invalid `ANTHROPIC_API_KEY`, zero credit balance.
- **Ask retrieval mode is not user-configurable.** Hardcoded to `hybrid` inside `retrieve()`; no UI switch. Deliberate — exposing the knob added noise without gain.
